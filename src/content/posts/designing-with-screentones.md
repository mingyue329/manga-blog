很多技术站点的问题不在于信息不够，而在于情绪几乎被清空了。大家都在追求“干净”“理性”“专业”，结果最后做出来的页面常常只是安全、平整、没有记忆点。

## 我想要的不是装饰，而是语气

网点、速度线、厚边框和偏移阴影在这个项目里都不是“点缀”，它们承担的是语气功能。它们告诉读者，这个页面不是某个默认模板，而是一张被明确设计过的纸面。

如果把这些元素随手乱加，它就会变成风格噪音；只有先决定它们分别表达什么，页面才会稳定。

## 我怎么拆视觉语言

我会先定义几条足够明确的规则：

1. **标题负责冲击感**：字号更大，字重更黑，允许有轻微倾斜。
2. **边框负责结构感**：重要区域用 4px 黑色边框，普通区域避免细灰线。
3. **网点负责密度变化**：不是为了填满空白，而是为了制造节奏。
4. **阴影负责“纸张层叠感”**：要么有力，要么不要。

## 真正有用的，不是一套效果，而是一套约束

当你开始把风格规则写成系统，设计就会突然变轻松。因为你不再需要为每个区域重新发明语言，你只是在已有语法里做组合。

| 元素 | 允许做什么 | 不允许做什么 |
| --- | --- | --- |
| 按钮 | 方角、粗边、强对比 | 柔和圆角、浅灰边框 |
| 卡片 | 黑边框、偏移阴影 | 模糊阴影、弱分隔 |
| 辅助背景 | 网点、浅纹理 | 无意义渐变堆叠 |
| 页面标题 | 极强对比、夸张排版 | 与正文同级的普通标题 |

## 为什么黑白反而更难

因为你失去了用颜色救场的机会。没有彩色之后，空间关系、层级、字重、留白和节奏都会被放大检验。黑白风格不是“少设计”，而是“设计的每一步都更不能含糊”。

> 当色彩退场之后，结构就必须自己站得住。

## 代码实现时我最在意什么

前端实现里，我最怕的是视觉语言被拆散在各个页面里。所以我会优先抽出这些可复用能力：

- 全局变量，例如纸张色、墨色、阴影偏移量
- 明确的按钮变体，而不是每页自己写按钮
- 通用卡片边框与阴影规则
- 少量但稳定的纹理 utility class

```ts
const cardShadow = '10px 10px 0 0 #111111'
const heavyBorder = '4px solid #111111'
```

这些东西一旦形成统一约束，页面就更容易保持一个完整人格。

## 收尾

视觉风格不是“让站点看起来不一样”这么简单，它更像一种持续讲话的方式。只要语气稳定，页面再扩展几个模块、再增加几条路由，也不会突然失去自己的样子。
