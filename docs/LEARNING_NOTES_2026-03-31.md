# 2026-03-31 学习笔记

## 1. 这次项目最值得学习的地方

今天这轮实现最重要的不是“把页面做出来”，而是把页面整理成后面还能继续扩展的结构。重点可以从下面几个角度理解。

## 2. 为什么用 `router + loader + service + page`

这套结构的核心目标，是把“路由参数”“数据获取”“页面渲染”拆开。

### 2.1 `router`

职责：

- 决定页面路径
- 决定哪个页面组件渲染
- 决定页面进入前要调用哪个 loader

例如：

- `/posts` 对应归档页
- `/posts/:slug` 对应详情页

### 2.2 `loader`

职责：

- 从路由里拿参数
- 调用 service 获取数据
- 把结果交给页面

它不负责写 JSX，也不负责拼复杂业务逻辑。

### 2.3 `service`

职责：

- 组织数据来源
- 控制本地内容 / 远程接口切换
- 补齐页面真正需要的字段

比如文章详情页里，service 做了这些事：

- 按 `slug` 找文章
- 读取对应的 Markdown 文件
- 计算阅读时长
- 生成上一篇 / 下一篇
- 生成相关文章

### 2.4 `page`

职责：

- 只关心如何渲染页面
- 不直接处理复杂数据拼装
- 尽量保持 JSX 结构清晰

这也是为什么详情页看起来很复杂，但真正的数据逻辑没有写在页面组件里。

## 3. 为什么正文要放在 `src/content/posts/*.md`

如果把正文直接写进 `ts` 文件，会有几个问题：

- 改文章要改代码
- 内容和页面结构混在一起
- 后续迁移到 CMS 或接口会更乱

单独放 Markdown 的好处：

- 内容编辑更直接
- 页面逻辑和文章内容分离
- 后面切到远程接口时，service 层更容易替换

## 4. `PostMarkdown` 组件为什么单独抽出来

不是因为它“代码很多”，而是因为它承担了统一样式规则的职责。

如果不抽出来，页面里会出现这些问题：

- 每个详情页各写一套 Markdown 样式
- 标题、引用、代码块风格慢慢漂掉
- 后期维护时很难统一调整

抽出来之后，好处是：

- 所有 Markdown 页面共享同一套排版规则
- 以后想改正文风格，只改一个组件
- 站点整体风格更稳定

## 5. 文章详情页为什么没有完全照搬 Stitch 稿

这是一个很重要的前端实现判断。

虽然详情页参考了 `stitch_about_me_articleDetails`，但没有原样照搬，原因是：

- 它的风格和现有站点不完全一致
- 如果强行照搬，会让详情页像另一个网站
- 当前项目更需要“全站统一”，不是“单页独立炫技”

所以最终做法是：

- 保留详情页该有的结构节奏
- 但视觉语言收回到全站已有的黑白漫画科技风格

这个判断在真实项目里很常见：

- 设计稿是参考
- 项目风格一致性是约束
- 最终代码需要在两者之间做平衡

## 6. 背景动效这次学什么

这次点阵背景不是简单铺一张图，而是做成了全局动态层。

这里可以重点理解两个点：

### 6.1 为什么会出现鼠标偏移

因为视觉层的位置和鼠标坐标的计算基准不一致。

解决思路不是“拍脑袋调偏移量”，而是：

- 让坐标基于背景容器自身计算
- 保证热点和视觉层使用同一坐标系

这是前端做交互效果时很常见的定位问题。

### 6.2 为什么 `header/footer` 要透明

如果顶层容器自己有实底色，就会把下面的背景效果盖住。

所以这次不是去单独给 `header/footer` 再写一套背景，而是：

- 让它们变成透明层
- 直接透出下面的全局背景

这样整站会更统一，也更好维护。

## 7. 这次实现里几个值得记住的习惯

### 7.1 把“会复用的规则”抽出来

例如：

- Markdown 样式
- 文章摘要类型
- 上一篇 / 下一篇卡片
- 相关文章卡片

### 7.2 把“未来要变的东西”留在 service 层

例如：

- 本地静态内容和远程接口切换
- 阅读时长计算
- 相关文章生成规则

### 7.3 页面组件尽量只负责渲染

这样 JSX 才不会越写越乱，也方便以后你自己改结构。

## 8. 你后面可以继续重点看什么

如果你要继续拿这个项目学习，建议按这个顺序看：

1. 先看 `router.tsx`，理解页面怎么挂上去。
2. 再看详情页的 `loader` 和 `service`，理解数据怎么进入页面。
3. 再看 `post-detail-page.tsx`，理解页面怎么把数据拼成布局。
4. 再看 `post-markdown.tsx`，理解统一样式收口的思路。
5. 最后看 `interactive-dot-background.tsx`，理解全局背景动效的实现方式。

## 9. 一句话总结

今天这轮工作的核心，不是“多写了几个页面”，而是把一个设计稿项目整理成了一个有清晰边界、可继续扩展、也适合学习的前端结构。
